angularjs

基本介绍

基本使用

1、单页面应用程序：多个页面重复应用的东西，不会被多次加载，只加载页面中可改变的内容
2、解析链接地址
【1】在后台输入 window.location 回车 ，打印出 可以获取链接地址内容的属性

【2】在后台创建 一个 a 标签 ，
var a=document.createElement(“a”);
a.href=”地址”；
a.属性就能获取到链接地址的内容

3、指令
【1.】 ng-app 在 ng 中被称之为一个指令
该指令的作用就是用来指定 ng 应用程序的入口范围的
只有作用了该指令的元素及该元素内部所有子节点都可以直接使用 ng 所有的规则特性
注意：一页页面只有一个 ng-app 入口标识
建议挂载到刚好够用的范围上，一般就是 body 即可
ng-controller=”LoginController” 绑定控制器函数
【2】ng-model 在 ng 中也被称之为一个指令
该指令一般用于和表单控件（input、text、checkbox、radio、select、textarea。。）进行模型数据绑定，
双向数据绑定：
这里就是将对象中的 foo 属性绑定到了 input 的 value 上
当 input 的 value 改变的时候，数据对象上的 foo 也跟着变
当 数据对象上的 value 发生改变的时候，把所有引用或者绑定了我 foo 属性的地方同步改变过去
【3】 ng-bind 可以解决表达式闪烁的问题
表达式如何使用，ng-bind 就如何使用

1. 引包
2. 设置 ng-app 入口标识

1. 当 ng 启动的时候，找到 ng-app
2. ng 自动创建一个对象（目前还看不到）
3. ng 发现 body 上有一个 ng-init 指令，该指令表示在对象中添加一个属性：foo = 'bar'

【4】控制器
ng-controller=”LoginController” 绑定控制器函数
控制器就是给局部的 HTML 添加一个数据模型作用域，
该内部的 HTML 只能访问自己所属的控制器上的模型数据成员

  1. 根据视图初始化模型成员
    1. 然后做对应的业务处理
      这个函数一般被称之为控制器函数：用来控制视图和数据模型的交互处理的
      用法：第一：在创建的时候，建议控制器函数的名字和普通函数区分一下，采用 Pascal 命名法 + Controller 用来加以区分
      第二：创建完毕之后，在 HTML 中通过 ng-controller 指令将控制器和对应的 HTML 做一个绑定处理，这样话作用了该控制器节点的 HTML 以及内部所有的子节点就都可以访问控制器中的数据模型成员了
      第三：控制器函数必须全局函数，而且参数名必须叫：$scope，否则直接报错

/**

  • 控制器函数：用来控制视图和模型数据的交互
  • 控制器可以为局部的 HTML 添加数据模型作用域，只作用于该 HTML 片段
  • 如何划分控制器？
  • 根据你的不同的业务功能去划分你的控制器
  • 如果一个页面全局控制器函数太多，会导致全局命名空间污染。或者控制器被覆盖的问题
  • 在 ng 1.3 之前，是可以写全局控制器的
  • 但是在 ng 1.3 之后，就不再有这种写法
    */

ng 提供了一种模块化解决方案，可以有效的解决全局控制器污染问题

 1. 创建模块
    注意：第二个参数一定要给一个空数组
          如果不传递第二个参数，表示获取一个模块
var app = angular.module('app', [])
module括号里面的 名字起得是什么，页面内 ng-app 就得等于啥

例如
angular.module(‘app’, [])
ng-app=”app”

// 2. 给模块添加控制器
app.controller('LoginController', function ($scope) {
  $scope.username = 'login'
})

app.controller('RegisterController', function ($scope) {
  $scope.username = 'register'
})

【5】
g-submit 作用域表单元素上
当表单提交的时候，会触发 submit 事件，然后执行对应的处理函数
只有两种情况会提交触发这个 submit 事件：

1. 表单中具有 submit 类型的 input 可以提交
  2. 表单中在某个表单控制上按回车也可以提交表单

【6】ng-class 指令专门可以解决样式的问题
ng-class=”{css类名: bool}”
当 bool 为 true，作用 css类
当 bool 为 false，取消 css 类
